# Prototype Report: Wire Grapple 2D

## Hypothesis
ショクワンダー式ワイヤーアクション（振り子ではなく目標点への直線引き寄せ）は、2D横スクロールで気持ちいい移動体験を作れるか？

## Approach
- `CharacterBody2D` + 4ステート状態マシン（NORMAL / WIRE_TRAVELING / WIRE_FAILED / WALL_CLINGING）
- ワイヤー移動中は `collision_mask = 0` で全衝突を無効化し、目標点への直線移動を保証
- 壁到達時は張り付き、床・天井到達時は通常状態に遷移
- 失敗時はワイヤーが伸びて戻るTweenアニメーションでフィードバック
- しゃがみによる当たり判定縮小（床最小射程制限付き）

## Result
- 直線引き寄せの基本挙動は実現できた
- 壁張り付き→ワイヤー連打による移動チェーンは機能する
- 手触りに改善の余地あり（速度・射程・重力感のチューニングが必要）
- 修正が必要だったバグ：到達判定タイミング、レイキャスト2重発射による貫通、Camera2Dジッター

## Metrics
- 実装イテレーション回数: 約10回
- 致命的バグ: 3件（到達判定・レイキャスト貫通・カメラジッター）→ 全修正済み
- Feel assessment: 「移動できる」は達成、「気持ちいい」はパラメータチューニング継続中

## Recommendation: PROCEED

直線引き寄せ方式はGodot 4 + Jolt Physicsで実装可能と確認。コアループは成立しており、手触りの課題はパラメータと演出（アニメーション・サウンド・カメラワーク）で解決できる範囲と判断。

## If Proceeding

- **アーキテクチャ**: 状態マシンは `PlayerState` リソースへの分離を推奨（スクリプト肥大化防止）
- **パラメータ**: 定数から `@export` 変数に移行しエディタから調整可能に
- **追加が必要な要素**: ワイヤーの視覚演出（エフェクト・太さ変化）、SE、カメラシェイク
- **推定工数**: M（1〜2スプリント）— コアロジックは判明済みのため、主にアート・音・チューニング

## Lessons Learned

- `move_and_slide()` の前後で到達チェックのタイミングが変わる（後が正確）
- ワイヤー移動中の `collision_mask = 0` は必須。物理エンジンに任せると経路上の障害物でブロックされる
- Camera2D をCharacterBody2Dの子にすると物理レートでジッターが発生する。兄弟ノードにして `_process` で追従させること
